【根本原因】

問題 1: **只計算了 2024-2025，但驗證時檢查所有年份**
  - 2020-2023 年：使用舊方法（需要 outstanding_shares，覆蓋率 7.42%）
  - 2024 年：使用新方法（taiwan_stock_per，覆蓋率 6.82%）← 反而更低！
  - 2025 年：使用新方法（taiwan_stock_per，覆蓋率 7.44%）

問題 2: **taiwan_stock_per 數據密度低於預期**
  - taiwan_stock_per: 2024 年 47,559 筆，242 個唯一日期
  - 203 支股票 × 242 天 = 理論 49,126 筆
  - 實際 47,559 筆 → 覆蓋率 96.8%（有些股票某些日期沒數據）

問題 3: **非財報股票占 stock_factors 的 91.3%**
  - stock_factors 總記錄: 3,487,238 筆
  - 有財報股票 (207 支): 303,756 筆 (8.7%)
  - 無財報股票 (2,135 支): 3,183,482 筆 (91.3%)
  
  → 即使 207 支股票 100% 覆蓋，總覆蓋率也只有 8.7%！

問題 4: **數據源限制無法突破**
  - FinMind 只提供 207 支主要上市櫃公司的財報
  - 其他 2,135 支（ETF、興櫃、指數等）無財報
  - 這是 FinMind API 的結構性限制

【為什麼 taiwan_stock_per 沒有幫助？】

理論推導：
  - Taiwan_stock_per 涵蓋 203 支，98.1% 覆蓋有財報股票 ✓
  - 但這 203 支只占 stock_factors 的 8.7% ✓
  - 即使 100% 覆蓋這 203 支，總覆蓋率也只有 8.7% ✓

實際結果：
  - 2024 年 PE 覆蓋率: 6.82% vs 理論上限 8.7%
  - 達成率: 78.4% (6.82% / 8.7%)
  - 差距原因: taiwan_stock_per 並非每個 (股票, 日期) 都有數據

對比舊方法：
  - 2020-2023 年 PE 覆蓋率: 7.42%
  - 舊方法達成率: 85.1% (7.42% / 8.7%)
  - **舊方法反而更好！**

【為什麼舊方法更好？】

舊方法（使用 outstanding_shares）：
  1. outstanding_shares 雖然只有 8.3% 有值
  2. 但這 8.3% 分佈在更多年份（FinMind 早期數據）
  3. 一次查詢就能覆蓋該股票的所有日期
  4. 所以覆蓋率達到 7.42%

新方法（使用 taiwan_stock_per）：
  1. taiwan_stock_per 只有 271,103 筆記錄
  2. 每個 (股票, 日期) 需要精確匹配
  3. 如果某天沒有 PER 數據，就算不出 PE
  4. 所以覆蓋率只有 6.82%（2024）

【Priority 1 目標可行性分析】

目標: 因子覆蓋率達 80%

理論上限分析：
  1. stock_price 有 2,342 支股票的數據
  2. 其中只有 207 支有財報 (8.8%)
  3. 價值因子（PE/PB）和質量因子（ROE/ROA）**必須有財報**
  4. 理論上限 = 8.8%

結論：**Priority 1 目標物理上不可能達成**

原因：
  - 目標設定時假設所有股票都有財報
  - 實際上 91.2% 的股票沒有財報（ETF、指數、興櫃等）
  - FinMind 數據源無法提供更多財報數據

【重新計算所有年份會有幫助嗎？】

假設重新計算 2020-2025 所有年份：
  - 2020-2023: 7.42% → 可能提升到 7.44%（使用 taiwan_stock_per）
  - 2024: 6.82% → 維持 6.82%
  - 2025: 7.44% → 維持 7.44%

整體覆蓋率：
  - 預期: 7.2-7.4%（略低於當前的 7.33%）

結論：**重新計算所有年份不會改善，可能還會稍微下降**。
