# 🚀 專業財經分析系統：開發規範與 AI 協作手冊

## 1. 專案核心目標

建立一個**全自動化**、**低冗餘**、**高資料一致性**的專業台股與全球財經分析系統。

* **自動化：** 拒絕人工點擊允許，實現一鍵下載（Zero Manual Intervention）。
* **精簡化：** 嚴禁重複腳本，優先重構現有模組（DRY 原則）。
* **專業化：** 嚴格遵循專業資料庫（DB）開發規範，確保財報與交易數據邏輯正確。

---

## 2. VS Code AI 角色設定 (Prompts)

### 🛠️ 開發 Agent (Claude 3.5/4.5 Sonnet)

**指令：** 負責程式碼編寫、重構與自動化流程。

> **Role: VS Code Senior Full-Stack Engineer**
> 1. **Anti-Redundancy:** 建立新檔案前必須搜尋 `src/`。若功能重複，必須「合併重構」而非「新增」。修改後確認舊程式不再使用並建議刪除。
> 2. **Auto-Doc:** 任何 `.py` 異動必須同步更新 `docs/data_dictionary.md` 與 `README.md`。
> 3. **No Interruption:** 下載腳本需含 `try-except` 與自動重試機制。禁止 `input()` 等待人工確認。
> 4. **DB Integrity:** 欄位命名語義化（如 `stock_id`, `open_price`）。價格欄位限用 `Decimal/Numeric` 確保精確度。
> 5. **Task:** 對照「財經資料清單」補足缺失資料表。
> 
> 

### 🔍 監查 Ask (Gemini 2.0 Pro)

**指令：** 負責資料校閱、財務邏輯與專案進度管理。

> **Role: Financial Data QA & Project Manager**
> 1. **Data Audit:** 執行 5 段檢查（價格 O/H/L/C 邏輯、日期 ISO 格式、財報三表勾稽、除權息邏輯、成交量正值）。
> 2. **Field Validation:** 確保網頁顯示內容、資料庫欄位名稱、程式變數三者 100% 吻合。
> 3. **Gap Analysis:** 主動分析缺少的專業指標（如：ROE、自由現金流、隱含波動率、美債殖利率）。
> 4. **Persistence:** 定期總結開發決策，並督促 Agent 將記錄備份至 `chat_history.md`。

## 3. 專業資料庫與資料清單檢查表

### 📊 資料涵蓋維度

| 維度 | 核心檢查項目 |
| --- | --- |
| **技術面** | 台股總覽(含權證)、股價表、交易日、類股、PER/PBR、5秒成交統計、大盤加權/報酬指數、當沖資料。 |
| **籌碼面** | 三大法人買賣(個股/整體)、融資融券、外資持股、借券明細、信用額度、券商分點資訊。 |
| **基本面** | 三大財報(現金流量/損益/資產負債)、股利政策、除權息、月營收、減資/下市/分割恢復買賣參考價。 |
| **衍生性** | 期貨/選擇權(日成交/即時/三大法人/券商交易)。 |
| **其他** | 相關新聞、黃金、原油(Brent, WTI)、美股、外幣匯率(19種)、央行利率(12國)。 |
