# 每小时自动下载流通股数 - 使用指南

**创建时间**: 2026-02-21  
**适用场景**: FinMind API 免费版（每小时配额重置）

---

## 功能特点

### ✨ 智能特性

1. **自动配额管理**
   - 检测 API 402 错误（配额耗尽）
   - 自动计算等待时间到下一个整点
   - 倒计时显示（每分钟更新）

2. **断点续传**
   - 自动跳过已下载的股票
   - 配额重置后继续下载剩余股票
   - 支持中断后重新启动

3. **完整日志**
   - 实时进度显示
   - 详细的成功/失败记录
   - 每小时统计报告
   - 最终完成总结

4. **安全机制**
   - 最大运行时间限制（防止无限循环）
   - Ctrl+C 优雅中断
   - 错误自动重试

## 命令参数

### 基本参数

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `--priority-list` | 使用优先股票列表（核心 50 支） | - |
| `--all` | 下载所有股票 | - |
| `--max-hours` | 最大运行小时数 | 24 |

### 范例

```bash
# 下载优先列表，最多运行 3 小时

    --priority-list \
    --max-hours 3

# 下载全部股票，最多运行 12 小时
python3 src/downloaders/hourly_outstanding_shares_downloader.py \
    --all \
    --max-hours 12
```

## 故障排除

### 问题 1: 一直显示 402 错误

**原因**: API 配额确实耗尽

**解决**:
- 等待程序自动倒计时
- 或手动中断（Ctrl+C），等到下一个整点后重新执行

---

### 问题 2: 程序卡住不动

**原因**: 网络问题或 API 响应超时

**解决**:
```bash
# 中断程序（Ctrl+C）
# 检查网络连接
ping api.finmindtrade.com

# 重新执行
python3 src/downloaders/hourly_outstanding_shares_downloader.py --priority-list
```

---

### 问题 3: 找不到优先列表文件

**错误信息**:
```
找不到优先列表: /Users/ming/Desktop/Stock/tw-stock-analysis/data/priority_stocks.txt
```

**解决**:
```bash
# 检查文件是否存在
ls -la data/priority_stocks.txt

# 如果不存在，重新创建（联系技术支持）
```

---

### 问题 4: API Token 无效

**错误信息**:
```
❌ 错误: 请设定 FINMIND_API_TOKEN 环境变量
```

**解决**:
```bash
# 检查 .env 文件
cat .env | grep FINMIND_API_TOKEN

# 手动设定
export FINMIND_API_TOKEN="your_token_here"
```
