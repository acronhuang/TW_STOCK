## 根本原因

### FinMind API 配额限制

**免费账户限制** (推测):
- API 调用次数: ~500-600 次/天
- 触发时机: 第 578 次请求
- 重置周期: 24 小时（次日重置）

**本次消耗**:
- 基本面数据下载: ~200 次 (dividend_detail, stock_price)
- 流通股数下载: ~578 次
- **总计**: ~778 次
- **结果**: 超出配额

### 股票排序问题

**taiwan_stock_info 排序**（按 stock_id 字母顺序）:
1. 0050-0999 (ETF，无资产负债表)
2. 1301-1599 (传产股)  ← **成功下载区间**
3. 2303-2499 (电子股)  ← **部分成功**
4. 2330-2454 (核心股票) ← **❌ API 配额耗尽时刻**
5. 2881-6000+ (金融/其他)

**结果**: 核心股票（2xxx 开头）恰好在 API 配额耗尽后。
