### 2. 驗證機制

#### 轉折點識別
```python
self.pivot_identifier = PivotIdentifier(order=5, threshold=0.02)
```
- 使用 scipy.signal.argrelextrema
- 窗口大小: 5（左右各5根K線）
- 價格變動閾值: 2%

#### 頸線檢測
```python
self.neckline_detector = NecklineDetector(tolerance=0.03)
```
- W底/M頭頸線自動識別
- 價格容忍度: 3%
- 自動檢測突破和量能確認

#### 支撐壓力識別
```python
self.sr_detector = SupportResistanceDetector(tolerance=0.02, min_touches=2)
```
- 價格聚類容忍度: 2%
- 最少觸碰次數: 2
- 強度評估

### 3. 信心度增強邏輯

**基礎匹配** (+5%):
- 12神招檢測到W底 + 頸線模組檢測到W底頸線
- 12神招檢測到M頭 + 頸線模組檢測到M頭頸線
- 頸線價格誤差 < 3%

**量能確認** (+3%):
- 已突破頸線
- 突破日成交量 > 前5日平均 × 1.5

**信心度上限**: 100%

## 📖 使用指南

### 基礎使用（自動驗證）

```bash
# 單股分析 - 自動享有雙重驗證
python3 pattern_recognition/multi_timeframe_scanner.py \
  --symbol 2330 \
  --timeframes D W M \
  --show-suggestion

# 全市場掃描 - 自動驗證所有信號
python3 pattern_recognition/market_multi_timeframe_scanner.py \
  --timeframes W M \
  --min-resonance 0.80 \
  --save-csv
```

### 驗證效果觀察

查看日誌輸出中的驗證資訊：
```
INFO - 時間週期 D: 找到 10 個形態信號 (頸線驗證: 1, 支撐: 1, 壓力: 0)
```

**解讀**:
- `頸線驗證: 1` - 檢測到1個W底或M頭頸線
- `支撐: 1` - 檢測到1個支撐位
- `壓力: 0` - 未檢測到壓力位


## ⚠️ 注意事項

### 1. 參數調整

整合後的預設參數已最佳化：
- 轉折點識別: order=5, threshold=0.02
- 頸線檢測: tolerance=0.03
- 支撐壓力: tolerance=0.02, min_touches=2

**建議**:
- 大型權值股使用預設參數
- 小型股可調整 threshold=0.03-0.05
- 趨勢明確時可調整 tolerance=0.02

### 2. 已知限制

**形態重複檢測**:
- 頭肩底可能檢測出多個重複信號
- 原因：patterns_12_masters.py 尚未實作去重
- 解決：未來版本將加入去重邏輯

**支撐壓力偵測**:
- 需要足夠的轉折點
- 交易量小的股票可能檢測不到
- 建議搭配其他技術指標

### 3. 效能影響

**整合前**:
- 單股掃描: ~1秒
- 市場掃描(50支): ~1分鐘

**整合後**:
- 單股掃描: ~1-2秒 (+0-1秒)
- 市場掃描(50支): ~1-1.5分鐘 (+0-30秒)

**結論**: 效能影響可接受，驗證價值遠大於時間成本
